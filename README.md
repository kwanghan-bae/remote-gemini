# 📱 Gemini CLI Remote Bridge V2 (내일 바로 사용하기 가이드)
### "출근하자마자 5분 만에 세팅하고 스마트폰으로 AI 개발을 시작하세요."

이 문서는 개발자가 아니어도 따라 할 수 있도록 가장 직관적으로 작성되었습니다. [클로드 코드 원격 제어](https://adim.in/p/remote-control-claude-code/)의 사용자 경험을 그대로 경험하실 수 있습니다.

## 🛠️ [중요] 시작 전 딱 3가지만 준비하세요

### 1. 비밀번호 설정 (`.env` 파일 만들기)
프로젝트 폴더 안에 `.env`라는 이름의 새 파일을 만들고 아래 내용을 복사해서 넣으세요.
```env
PORT=3000
AUTH_TOKEN=내가_정할_비밀번호 (예: super-secret-123)
```
> **Tip!** Gemini CLI 경로는 서버가 실행될 때 **자동으로 탐색**하므로 따로 설정할 필요가 없습니다.
> **주의!** `.env.example` 파일을 참고하셔도 좋지만, 직접 `.env` 파일을 만드셔야 서버가 작동합니다.

### 2. 서버 실행하기 (내 PC)
터미널(또는 CMD)을 열고 아래 명령어를 순서대로 입력하세요.
```bash
npm install     # 필요한 프로그램들 설치 (1회만 수행)
npm run build   # 프로그램을 실행용으로 변환
npm start       # 서버 시작!
```

### 3. 화면에 나오는 주소 확인하기
서버가 켜지면 아래와 같이 파란색 화살표가 있는 주소가 나옵니다.
`👉 http://192.168.0.10:3000/?token=내가_정할_비밀번호`
이 주소를 스마트폰 크롬 브라우저에 그대로 입력하세요.

---

## 📲 스마트폰에서 최적의 경험을 얻는 팁

### 1. '앱'처럼 설치하세요 (PWA)
- **로컬 접속 팁 (설치 메뉴가 안 보일 때)**: 
  스마트폰 크롬은 보안상 로컬 IP(`192.168.x.x`)에서 '앱 설치'를 숨깁니다. 이를 풀려면:
  1. 크롬 주소창에 `chrome://flags/#unsafely-treat-insecure-origin-as-secure` 입력
  2. **Enabled**로 변경 후, 아래 칸에 내 주소(예: `http://192.168.45.28:3000`) 입력
  3. 크롬 재시작 후 접속하면 메뉴에 **'앱 설치'**가 나타납니다.
- 설치 후에는 주소창 없이 **전체 화면**으로 터미널을 쓸 수 있어 진짜 앱처럼 느껴집니다.

### 2. 하단 툴바를 적극 활용하세요
- 스마트폰 키보드에는 `ESC`나 `Tab` 키가 없습니다. 
- 제가 터미널 하단에 만들어둔 **특수 키 버튼들**을 누르면 `Gemini`에게 명령을 내리고 답변을 승인(Y/N)하는 과정이 훨씬 빨라집니다.

### 3. 세션 유지 (Persistence)
- 폰에서 잠깐 다른 앱을 쓰거나 화면을 꺼도 작업이 끊기지 않습니다. 
- **5분 이내**에 다시 접속하면 하던 작업 그대로 이어집니다.

## 🏗️ 전체 동작 구성도 (Data Flow)
```text
[ 내 스마트폰 ] <--- WebSocket (실시간) ---> [ 내 PC 서버 ] <--- PTY (가상터미널) ---> [ Gemini CLI ]
      ↑                                           ↑                                      ↑
(가상 키보드/탭)                         (비밀번호 인증/보안)                       (AI 연산/파일제어)
```

이 브릿지는 로컬 터미널을 그대로 '중계'하므로 모든 기능이 로컬과 100% 동일하게 작동함을 보장합니다.
